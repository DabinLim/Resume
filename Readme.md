        <br>


    - 개발 난항 및 해결방법
        - 웹에서는 모달로 띄우던 카드 상세정보를 모바일는 페이지처럼 활용하는 과정에서 편의를 고려해 뒤로가기 버튼을 만들어두었으나 실제 유저가 사용했을때는 디바이스 자체의 뒤로가기 버튼을 사용하여 페이지가 모달을 띄우기 이전의 페이지로 넘어가는 문제가 있었습니다<br> 다수의 유저들이 불편함을 느껴 해결방안을 찾기 시작하였고 innerWidth를 활용하여 모바일에서 클릭 시 페이지 이동을 하는 방법과 resize 이벤트 구독을 통해 클릭 없이 페이지 이동을 하는 두가지 방법을 생각하였습니다.<br> resize 이벤트를 카드가 있는 페이지마다 구독할 경우의 성능 저하 문제를 고려하여 innerWidth를 활용하여 일정 크기 이하에서는 페이지 이동을 하게 수정하였으며 웹사이트에서 모달을 띄운 후 화면 크기를 줄였을때 경우에 대비해 안내문구와 버튼을 통해 페이지 이동을 할 수 있게 구현하였습니다.

        - moment.js를 리덕스 state에 저장했을때 날짜가 변경되었음에도 해당 state를 참조하는 컴포넌트가 리렌더링이 되지 않는 문제가 있었습니다.<br>
        해당 문제를 해결하기 위해 format한 값을 또 다른 state에 새로 할당하여 리렌더링이 필요한 경우(날짜 이동을 해 책장이 업데이트 되야 하는 경우)에는 해당 state를 참조하도록 하여 문제를 해결하였습니다.

        - 짧은 시간 안에 잦은 디자인 변경과 피드백 반영으로 chrome extension을 출시하고자 하였음에도 투자 할 시간이 부족하였습니다.<br>
        최대한 빠른 시간안에 원하는 기능을 구현한 extension을 출시하기 위해 익숙한 React를 활용할 수 있는 방법을 찾아보았고 기존의 사용하던 코드를 활용하여 효율적으로 chrome extension을 출시하였습니다.

        - 카드 상세정보의 경우 여러 페이지에서 사용을 하다보니 3명의 개발자가 각각 필요한 기능을 추가하게 되면서 코드의 길이가 길어지고 서로 모르는 코드들이 많아져 나중에는 작은 부분 하나를 수정하더라도 오랜 시간이 걸리는 큰 불편함을 겪었습니다.<br>
        해당 부분을 해결하기 위해 좋아요 버튼, 댓글창, 주제 버튼 등 더 작은 단위로 쪼갤 수 있는 부분을 찾아 따로 컴포넌트를 만들었으며 이러한 과정을 통해 코드의 양을 줄여 가독성을 높여 해결하였습니다.

        - 웹 메인 페이지에서 3장의 카드가 번갈아가면서 돌아가는 효과를 주기 위해 라이브러리를 사용하였지만 디자이너가 요구하는 대로 화살표 버튼을 바꾸기 위해 라이브러리 내부에 접근하기가 어려운 문제가 있었습니다.<br>
        개발자 도구를 통해 수정해야 할 부분을 찾아보았지만 원하는 대로 수정이 어려웠으나 구조를 살펴보던 중 pre-seleced, selected, pro-selected 세 부분으로 나누어 서로 순서를 바꿔주는 로직을 짜면 라이브러리 없이 구현이 가능하겠다는 생각이 들었고 생각대로 진행하여 디자이너의 요구사항을 맞출 수 있었습니다.

        - 여러 페이지에서 무한 스크롤 컴포넌트를 재활용하여 사용하였습니다<br>
        전체 페이지인 window를 기준으로 구현하는 경우, ref 값을 통해 해당 부분을 기준으로 구현하는 경우, 모달 내에서 구현하는 경우 이렇게 다양한 기준을 맞추기 위해 props 값에 따른 분기처리를 통해 대부분의 페이지에서 사용이 가능한 컴포넌트를 만들었습니다.
